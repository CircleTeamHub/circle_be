# Circle Admin Web 实施计划

## Summary

建设独立的 `circle_admin_web` 管理后台，用于管理员处理举报、封禁/解封/删除用户、查看轻量系统状态。项目独立于移动端和 NestJS 后端，云上作为单独 Docker 容器部署，并与后端处于同一 Docker Compose 与内网 network。

## Scope

- 前端：Vite + React + TypeScript + Ant Design + React Router + TanStack Query。
- 后端：补齐用户 status 筛选、自我封禁/删除保护、状态变更业务日志、admin 专用登录与 refresh token audience。
- 部署：`admin_web` 独立容器，Nginx 提供 SPA 和 `/api/` 内网反代，边缘代理把 `admin.<domain>` 指向 `admin_web:80`。
- 安全：Cloudflare Access/VPN/IP allowlist 作为第一层，后端 `JwtGuard + AdminGuard` 作为第二层。

## Backend Tasks

1. `GET /api/v1/user` 支持 `status?: UserStatus` 筛选。
2. `PATCH /api/v1/user/:id/status` 支持 `reason?: string`，最长 500 字。
3. 禁止管理员把自己的状态改为 `BANNED` 或 `DELETED`。
4. 成功变更用户状态后写入结构化 business log：`admin_user_status_changed`，包含 `actorId`、`targetId`、`oldStatus`、`newStatus`、`reason`。
5. 新增 `POST /api/v1/auth/admin/login`，只允许 `role=ADMIN && status=ACTIVE` 的账号登录。
6. 新增 `POST /api/v1/auth/admin/refresh`，只接受 `audience=ADMIN` 的 refresh session。
7. JWT access token 增加 `aud`：普通 App token 为 `APP`，后台 token 为 `ADMIN`。
8. `AdminGuard` 同时校验 `role=ADMIN` 和 `audience=ADMIN`。
9. `RefreshToken` 增加 `audience` 字段，默认 `APP`。

## Admin Web Tasks

1. 登录页调用 `/auth/admin/login`，token 存入 `sessionStorage`，随后调用 `/auth/me`。
2. 只有 `role=ADMIN` 且 `status=ACTIVE` 的账号可进入后台。
3. API client 自动注入 token；遇到 401 时调用 `/auth/admin/refresh` 一次并重试，失败则清空 session。
4. App shell 提供 Dashboard、举报审核、用户管理、系统状态四个入口。
5. Dashboard 展示待处理举报数、用户总数、封禁用户数、Outbox failed 摘要与外部运维链接。
6. 举报审核支持 `PENDING/APPROVED/REJECTED` tabs、详情抽屉、通过/驳回和审核备注。
7. 用户管理支持 accountId 搜索、status 筛选、封禁/解封/删除，危险操作需要 reason。
8. 系统状态展示 `/outbox/health` 和 Grafana/Sentry/Uptime Kuma/Alertmanager 链接。

## Deployment Tasks

1. `circle_admin_web/Dockerfile` 使用 Node build stage 与 Nginx runtime stage。
2. `circle_admin_web/nginx.conf` 支持 SPA fallback，并把 `/api/` 反代到 `http://circle_be:3000/api/`。
3. `docker-compose.prod.yml` 增加 `admin_web` service，使用 `../circle_admin_web` 作为 build context。
4. 边缘代理推荐使用 `deploy/Caddyfile.admin`，用 `API_DOMAIN`、`ADMIN_DOMAIN`、`ACME_EMAIL` 环境变量配置域名。
5. 生产环境 `ALLOWED_ORIGINS` 增加 `https://admin.<domain>`。

## Verification

- Backend user/admin API tests: `npm test -- user/__tests__/user.service.spec.ts user/__tests__/user.controller.spec.ts auth/__test__/refresh-token.service.spec.ts auth/__test__/auth.service.spec.ts auth/__test__/auth.controller.spec.ts auth/__test__/auth.strategy.spec.ts guards/__tests__/admin.guard.spec.ts --runInBand`
- Frontend tests: `npm test`
- Frontend typecheck: `npm run typecheck`
- Frontend build: `npm run build`
- Deployment: `docker build` admin web image
- Deployment: `docker compose -f docker-compose.prod.yml config`

## Rollout

1. 本地启动后端和 `circle_admin_web`。
2. 用 admin 账号验证登录、举报列表、用户列表、状态变更、系统状态。
3. 云服务器部署 `admin_web` 容器。
4. 配置 `admin.<domain>` 的 Cloudflare Access/VPN/IP allowlist。
5. 完成 smoke test 后开放给运营/审核人员。
